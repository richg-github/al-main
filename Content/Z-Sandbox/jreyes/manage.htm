Manage Notifications

The Notifications page in the Alert Logic console offers a consolidated view of your notifications. The list includes all notifications created from your account. You can create, view, and manage notifications of all types from this page.

The main panel on the right summarizes your notifications and organizes them in these tabs:

  • Alerts—Lists alert-based notifications such as incidents and log correlation observations. You can create a notification, and you can filter and search the list.
  • Scheduled Actions—Lists notifications for scheduled activities, such as scheduled reports. You can create a schedule, and you can filter and search the list.

To access the Notifications page, click the Settings icon (), and then click Notifications.

Create any type of notification

One way to create a notification of any type is directly from the Notifications page.

From the Alerts tab, you can click the add icon () to create these types of notifications:

  • Observations—Select this type to create a notification for observations generated by log correlation rules. For more information about setting up the notification, see Start creating a correlation.
  • Incidents—Select this type to create a notification for incidents that meet specific criteria, such as threat level. For more information about setting up the notification, see Create an incident notification.

From the Scheduled Actions tab, you can click the add icon () to create a Scheduled Report notification. Select this notification type if you want to schedule a report to run periodically and subscribe recipients to receive a notification when the report is generated.

Because you need to set up the report you want to schedule, the Scheduled Report option leaves the Notifications page and opens the Reports page.

For more information about setting up a scheduled report notification, see Create a report schedule and notification.

about setting up the notification

  1. In the Notifications page, click the Alerts tab, and then click the add icon ().
  2. Select the type of alert notification you want to create.
  3. Set up the notification in the page that appears.

Filter the notifications list

The Notifications page displays all notifications for your accounts and the accounts you manage. When you display active, inactive, or deleted notifications, you can apply additional filters to those notifications.

Filters vary according to the notification type. As you make selections, the list of available filters changes further.

Filters available for all notification types include:

  • Type
  • Managed Accounts
  • Recipients

If you select the Type filter, and then select Incidents, the following filters are also available:

  • Threat level
  • Classification
  • Detection Source
  • Deployment

If you are viewing notifications on the Scheduled Actions tab, the Customers filter is also available.

If you select the Type filter, and then select Scheduled Reports, the Report Category filter is also available.

To filter the notifications list:

  1. In the left navigation, click the notification status of interest:
    • Active
    • Inactive (notification is saved but not turned on)
    • Deleted
  2. Click any of the filters to further narrow the list. Available filters vary according to the notifications in your environment and filters you select.
  3. To search for a filter, type a filter value in search filters.
    4. For example, type "attack" to quickly find notifications about incidents classified as application attacks.
  4. To clear filters and start over, delete text typed in search filters (if applicable) or click CLEAR ALL FILTERS.

Group and sort the notifications list

Alert Logic groups your alert-based notifications by threat level and your scheduled actions by schedule type. Notifications are sorted by date within each grouping. You can group and sort the notifications by other criteria. The options available vary according to the notification type and the filters applied.

To organize your notifications:

  1. To change the grouping, click Group by, and then click the option you want. Available options match the filters listed in the left panel.
  2. To change the way the list of alert notifications is sorted within each grouping, click Sort by, and then click the option you want. Available options include Name, Create Time, and Updated Time. Available options include Date and filters listed in the left panel.

Search the notifications list

You can use the search bar to filter the list to include only notifications that contain specific words in important fields, like the notification name.

You can use the search bar to filter the list to include only notifications that contain specific search words. You can search by notification name, recipients, and filter names.

View details and manage notifications

You can view the details about a specific notification. The detail view includes information such as:

  • Date a notification that met the criteria was last sent
  • Notification creation and last modification date
  • Notification criteria
  • Subscribed users or integration
  • Delivery options

From the detail view, you can delete or edit the notification. Examples of editing capabilities include:

  • Change the notification status from active or inactive
  • Change the notification criteria (accounts and filters)
  • Subscribe or unsubscribe users or an integration
  • Change delivery options

You can also access the list of events (the Incident List, for example) for the specific notification type, with the list filtered the same as the notification criteria.

In the case of a scheduled report notification, you can view the interactive report and the list of reports previously generated by the schedule.

To view and manage a specific notification from the detail view:

  1. Click View next to a notification to open the detail view. You can do any of the following:
    • To delete the notification, click the Delete icon. Deleted notifications are moved from Active to Deleted status.
    • To edit the notification, click the Edit icon. The notification creation page appears, and you can change the settings.
    • To open the list of events of that type with filters populated, click the View icon.
    • To open a list of reports generated by the notification (applies to scheduled reports only), click the Artifacts icon.
  2. When you are finished viewing notification details, click Hide.

To view details about a notification:

In the Notifications page, click View next to the notifications you want to see. When you are finished viewing notification details, click Hide.

To delete a notification:

In the Notifications page, to the right of the notification you want to delete, click View. Toward the bottom of the detail view, click the DELETE icon .

Deleted notifications are moved to Deleted status.

To edit a notification:

  1. In the Notifications page, to the right of the notification you want to edit, click View.
  2. Toward the bottom of the detail view, click the EDIT icon.
  3. In the editing page, change any of the settings.

To open a list of incidents corresponding to the notification criteria:

In the Notifications page, to the right of an incident notification, click View.Toward the bottom of the detail view, click the View Incidents icon.

To view the interactive report related to a schedule:

In the Notifications page, to the right of the report schedule, click View.Toward the bottom of the detail view, click the VIEW REPORT icon.

To open a list of reports generated by a schedule:

In the Notifications page, to the right of the report schedule, click View. Toward the bottom of the detail view, click the VIEW ARTIFACTS icon.

Notifications listed

The Notifications page lists all notifications created from your customer account.

For a managing (parent) account, the list includes notifications for:

  • Managing account
  • Managed (child) accounts, if the notification was created from the parent account

In a managed account, the notifications list:

  • Includes only notifications created from that account
  • Does not include notifications created from a managing account
  • Does not include notifications created from another managed account

About managing notifications and subscriptions of others

You can manage the notification settings of others—including subscribing other users or an integration—only if your user account has one of the following user roles:

  • Administrator
  • Owner
  • Power User
    • Power users cannot manage the notifications and subscriptions of others in managed accounts unless they have a Power User role in the managed account.

Users maintain the ability to modify any changes you make to their notifications.

Notifications belong to the customer account where they are created. If you manage other accounts and want to subscribe users in a managed account, you need to switch to the managed account level.

You can set up a user account and designate it as "Notification Target Only," which means it exists only for email notifications. You can use this notification target to receive and store notifications for later review, or use a distribution list as the contact email so multiple people get the emails. For more information see Create a user account as a notification target.

Parent & Child Accounts (from https://algithub.pd.alertlogic.net/alertlogic/user-experience/tree/master/user-experience-deliverables/notifications/notifications-v4)

This section explains some of the interaction logic between parent and child accounts.

Use case: A parent account wishes to receive notifications about incidents generated in a child account. The parent account does not want the child account to be notified about the incidents that are generated.

In this use case, a user in the the parent account creates an incident alert and lists the child account in the "Accounts" section. The user also lists user/s of the parent account as recipients for Notifications that are generated. This alert would exist in the parent account and not the child account. This also means users in the child account can't get notifications for this alert, as it lives in the parent account and the child account has no permissions to see it.

Also in this example, the "View Incidents" button in the bottom left of the expanded Incident Alert card would not be displayed for users in the parent account. This is because the action of this button would direct the user to the Incidents console, displaying all of the incidents associated with that alert. Because these incidents exist in the child account and not the parent account, the button is not applicable here and therefore not displayed.

This interaction logic applies to all Notification types, not only Incident Alerts.
