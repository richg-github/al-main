Configure PagerDuty Connection

A PagerDuty connection securely stores reusable authentication credential and URL path information for integrations with PagerDuty.

Complete the following steps to configure a connection to PagerDuty:

  1. Generate your PagerDuty routing key
  2. Create the PagerDuty connection from the Alert Logic console
  3. Use the connection in templated connections and playbooks

Generate your PagerDuty routing key

To generate the PagerDuty integration service routing key, see the PagerDuty documentation.

Create the PagerDuty connection from the Alert Logic console

After you generate your PagerDuty routing key, the next step is to create the PagerDuty connection in the Alert Logic console.

To create a PagerDuty connection:

  1. In the Alert Logic console, click the navigation menu icon (), click Configure, and then click Connections.
  2. On the Connections page, click the add icon (), and then click PagerDuty.
  3. On the Create a PagerDuty Connection page, type a descriptive name for the connection—for example, "PagerDuty Connection".
  4. In Base URL, use the default Base URL.
  5. In Routing Key, paste the routing key you generated.
  6. Click SAVE.

Use the connection in templated connections and playbooks

After you save the connection target, the last step is to set up a connector or automated response playbook. For more information on connectors, see Connectors Configuration Guide. For more information on automated response, see Get Started with Automated Response (Beta).

After you save the connection, the last step is to set up a templated connection or automated response playbook. For more information on templated connections, see Templated Connections Configuration Guide. For more information on automated response, see Get Started with Automated Response.

Manage connections

You can view the list of connection targets and edit or delete an existing one. For more information, see Manage Connection Targets.

You can view the list of connections and edit or delete an existing one. For more information, see Manage Connections.
